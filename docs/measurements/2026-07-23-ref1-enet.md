# reference campaign stage 1 — session 1 Pre-registration(enet + envbase)

[ADR-0006](../adr/0006-reference-campaign-v1.md) の session 1。単発で回して
pipeline と時間・コスト見積もりを検証してから以降の boot に進む。

## 契約

- 予算 1.5h / 打ち切り 2h(boot 開始からの dispatch deadline)
- アウトプット: enet wan 3 cell(auth-s1000 / auth-s4000 / room)+
  envbase(wan)の screening 境界と confirmatory 判定
  (FROZEN / MORE_BLOCKS / INCONCLUSIVE)、見積もり vs 実績の host-hour
- fleet: c8g.16xlarge spot ×4(us-west-2)、bundle = dispatch 時点の
  master HEAD(clean)
- queue: `scripts/fleet/queues/ref1-enet`
  (`ref-genqueue.sh transports/enet.json queues/ref1-enet --with-envbase`)

## 手順

1. `campaign.sh` boot(boot gate: doctor + calibration + anchor
   fleet median ±10%)
2. class-mapping probe: host 1 台で
   `CONFIG=scripts/fleet/queues/ref1-enet/probe-enet.json` を
   `run-class-mapping-conformance.sh` で実行。enet 6 case PASS +
   Promotable で clean 証跡へ昇格。FAIL / INVALID なら測定へ進まず打ち切り
3. screening 4 job を dispatch(enet-scr-auth1000 / auth4000 / room、
   envbase-scr)
4. 各 cell の境界から `confirmatory-genqueue.sh <capacity.json>
   <confirm-*.tmpl.json>`(±10% 窓 × seed 1–3)→ dispatch
5. `confirmatory-analyze.sh` で停止規則を判定。MORE_BLOCKS の cell は
   seed 4–5 を追加(打ち切り時刻まで)
6. 回収 → 本 doc に結果追記 → `docs/catalog.md` 初版 → PR

## 判定・gate(全て凍結値、後出し変更なし)

- stopping rule: 連続 3 block 全幅 ≤5%、上限 5 block、未達 INCONCLUSIVE
- drift gate: delivery 前後差 ≤0.010 / staleness p99 前後比 ≤1.10 倍
  (block 前後の raw_udp envbase c128 baseline)
- doctor: job 直前に再生成(campaign.sh、15 分 freshness)
- 破断はゴール指標(到達率・鮮度・期限)のみ(ledger #29)

## pipeline 検証項目(session 1 固有)

- [ ] preset 指名 sweep が fleet で走り、results/capacity に preset_hash が入る
- [ ] job 直前 doctor 再生成で reference mode の config load が通る
- [ ] rig path(`$RIG` 相対 path)が host 側でも解決する
- [ ] confirmatory-genqueue → dispatch → analyze の一連が campaign summary で回る
- [ ] probe config(`probe-enet.json`)が fleet の CPU layout で走る
- [ ] 見積もり(~6 host-hour / ≈$8)vs 実績を記録し ADR-0006 の較正に使う

## 期待

- enet auth4000: pilot で ≥512(range censored)。上限 2048 で censoring 解消

## 結果(2026-07-24 打ち切り — spot 全滅)

attempt 1(campaign `ref1-enet-20260723`)は probe 段階で計測器バグを検出し
fleet ごと破棄(~$2.5)。attempt 2(`ref1-enet-20260723b`、bundle fa78721)は
probe 通過 → screening 3/4 完了後、**4 ホスト全てが同時間帯に spot 回収され
打ち切り**。累計 ≈11 host-hour ≈ $13。

### 検出・修正した計測器バグ(いずれも root cause 確認 → 修正 push 済み)

1. Graviton rig が reference 検証を通れない(governor 要求の x86 前提。
   doctor.go — commit fa78721)
2. bundle 展開ツリーで非 root unit が results-v2 を書けない(boot.sh 同)
3. **irqbalance が Ubuntu 既定で active** — isolate の IRQ 退避を随時崩し、
   doctor irq_cpu_isolation が flap(boot.sh で無効化 — commit f142a77)
4. conformance の doctor が setpriv で非 root 実行され、IRQ steerable 判定が
   判定不能 → 常に FAIL(root 実行に修正 — commit 739386e)
